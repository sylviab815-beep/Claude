# Hook Lab — Write & Audit FB Ad Hooks

A hook is the **first piece of text a reader processes** on a book ad — the line
that decides whether they keep paying attention. This skill does two jobs:
**GENERATE** fresh hooks from a book, and **AUDIT** a hook the author already
wrote. The hook framework, the four types, and the failure modes are shared
across both modes.

The full framework, example bank, and scoring rubric live in:
- `references/hook-framework.md` — the four types, failure modes, format & craft rules, one-two-punch, show-don't-tell conversions, example bank
- `references/audit-rubric.md` — the 6-point scoring rubric used in AUDIT mode

Read `references/hook-framework.md` at the start of EITHER mode. Read
`references/audit-rubric.md` as well when in AUDIT mode.

---

## Mode detection

Decide before doing anything else:

- **AUDIT** — The author pasted a hook / ad line and wants feedback, a score, or
  a fix. ("Audit my hook," "is this good," a quoted line on its own.)
- **GENERATE** — The author wants new hooks, gives book details, or asks for
  "hooks for my book / image / video." No existing hook to grade.
- **Unclear** — Ask one question: *"Do you want me to write new hooks for your
  book, or score and fix a hook you've already written?"* Then proceed.

---

## Shared: output format

Ask once, at the very start, before anything else:

> How do you want this delivered?
> **A) In chat** — markdown, paste-ready (default if they don't care)
> **B) Branded Word doc (.docx)** — formatted and downloadable

Carry the answer through the whole run; don't re-ask. For B, see "Branded .docx
output" at the bottom.

---

## Shared: the four hook types (always cover these)

Every hook is one (or a blend) of four types. Full definitions + examples in the
reference file; the short version:

1. **Curiosity gap** — an open or implied question with no obvious, trivial answer.
2. **Emotional resonance** — triggers a feeling the audience instantly recognizes.
3. **Provocative statement** — a bold/controversial line that halts the scroll.
4. **Story fragment** — a snippet that implies a much bigger story.

Default ranking when you must pick one: the **curiosity gap** generally converts
best — *but only if it matches the book.* Always match the hook's promise to the
real reading experience.

# GENERATE MODE
### Draft fresh hooks from a book

## G1 — Intake

Gather (pull what you can from the conversation; ask only for what's missing, in
one message):

1. **Book title** (and series, if any)
2. **Genre / subgenre** — drives everything
3. **Premise** — 1–3 sentences, or top tropes
4. **Character-focused or plot-focused?** — if unsure, infer from genre
   (military sci-fi → plot-focused; slow-burn romance → character-focused) and
   state your assumption
5. **Format wanted** — image hooks, video hooks, or both (default: both)
6. Optional: a known **comp** (book / movie / show) and any **standout review line**

## G2 — Set the spine

Before writing, lock two decisions and say them out loud:
- **Character vs. plot focus** (what the hooks will foreground)
- **Which hook types fit this book** (some genres lean curiosity/story-fragment;
  romance often leans emotional resonance). Plan to cover at least 3 of the 4.

## G3 — Write the hooks

Produce, by default, the **homework set**:
- **3 image hooks** — single-line, readable in one breath, each a different type
- **3 video hooks** — at least one built as a **one-two punch** (setup card →
  payoff card) with the first beat landing before ~2 seconds

For each hook give:
- The hook line(s) (for video, show the card breaks)
- Its **type** (curiosity / emotional / provocative / story fragment)
- A one-line **image/visual note** (what the picture should show — and where the
  character should look, since their eyeline should lead toward the text)
- For video one-two punches, the **bridge word** between cards (never on the
  opening line)

Every line must pass the GENERATE checklist in G4 before you present it.

## G4 — Self-check every hook (fix before showing)

Run each hook against the failure modes from `references/hook-framework.md`:
- [ ] **Shows, doesn't tell** — it doesn't just describe the image or the book
- [ ] **Not a sales pitch** — no e-commerce / "buy now" energy in the hook line
- [ ] **Not generic** — couldn't be slapped on half the books in the genre
- [ ] **No context debt** — no unexplained names; uses roles (the princess, the
      cartel) not proper nouns the reader hasn't met
- [ ] **No insider jargon** (unless deliberately targeting super-fans)
- [ ] **One-breath test** — readable in a single breath
- [ ] **Matches the book** — the promise fits the real reading experience

Rewrite anything that fails. Don't hand the author a hook that needs fixing.

## G5 — Deliver

Present the 3 image + 3 video hooks grouped and labeled by type, with the visual
notes. Add a short line noting which one you'd test first and why. Offer to
expand any into a full ad (headline + primary text) or to AUDIT a variation.

# AUDIT MODE
### Score and rewrite an existing hook

## A1 — Intake

Get the **hook text** (required). Ask for genre and image/video placement if not
obvious — but if the author just pastes a line and wants a quick read, infer
genre and proceed; don't stall.

## A2 — Score it

Read `references/audit-rubric.md` and score the hook on the **6-point rubric**
(0–2 each, /12 total):
1. Hook type clarity (is it clearly one of the four?)
2. Show vs. tell
3. Specificity (not generic)
4. Accessibility (no name-soup / jargon / context debt)
5. One-breath readability
6. Book match (promise fits the genre/experience)

Give the score, a one-line verdict band (see rubric), and call out the single
biggest problem first.

## A3 — Diagnose

For each point that lost marks, name the exact issue in one sentence and quote
the offending words. Be specific ("'everything changes' is generic — it fits any
book") not vague.

## A4 — Rewrite

Provide **2–3 stronger rewrites**, each labeled with its hook type, each passing
all six rubric points. If the original is image text, also suggest a video
one-two-punch version, and vice versa. Add a one-line visual note where relevant.

## A5 — Deliver

Score + verdict, the diagnosis, the rewrites. Offer to generate a full set
(GENERATE mode) or build the winning rewrite into a full ad.

---

## Hard rules (both modes)

- The hook is the FIRST text the reader sees — never lead with a sales pitch.
- Keep hooks to one breath. If it needs a second breath, it's too long.
- Show, don't tell — the image carries the "what genre," the text adds intrigue.
- Use roles, not unfamiliar proper names, in the hook.
- Match the hook's promise to the actual book. Never over-promise pace/heat the
  book doesn't deliver — it earns one-star reviews.
- Cover at least 3 of the 4 hook types in any GENERATE set.
- Don't duplicate the image hook in the headline (mention this if the author
  asks about headlines).

## Branded .docx output (option B)

Match the classroom house style used by the author's other ad skills.

**Palette (deep navy):** primary `1C3A5E`, primaryLight `DDE8F4`, promptBg
`EEF4FB`, tipBg `FFFBEC`, gold `C8A200`, tipText `5C4A00`.

**Structure:**
1. Navy title block — small kicker "GENRE & TROPE AI CLASSROOM", large "FB Ad
   Hooks" (or "Hook Audit"), sub-line "[Book Title]".
2. **GENERATE:** "Book at a Glance" box (genre, focus, format), then an Image
   Hooks section and a Video Hooks section — each hook in a clear box with its
   type label and visual note. End with "Which to test first."
3. **AUDIT:** the original hook in a box, the score table (/12) with the verdict
   band, the diagnosis, then the rewrites each in their own labeled box.
4. A small tip box: one-breath rule + show-don't-tell reminder.

Build with Node + the `docx` package; validate with the docx skill's
`validate.py`; save to the workspace folder as `Hooks - [Book Title].docx` (or
`Hook Audit - [Book Title].docx`) and present with `present_files`. If the docx
skill isn't available, fall back to chat markdown and say so.
