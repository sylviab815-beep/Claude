# Amazon A+ Content

## Core Rule

Treat A+ content as a retail conversion asset, not a decorative gallery. Build modules that quickly prove genre fit, story promise, series credibility, and reader payoff while staying consistent with the final cover and Amazon's book detail page context.

If the ebook cover is final but the manuscript is not, create a **provisional visual strategy** and clearly mark any story copy as draft. Prefer final copy only after the manuscript and blurb are complete or mostly locked.

## Intake

Collect or infer:

- Book title, author/pen name, series name, series order, formats, and whether the page is for one book, a series, or a box set.
- Genre, subgenre, heat/spice level if romance, age category, faith/clean constraints, and comparable authors or books.
- Final cover or cover direction. If no final cover exists, keep visual recommendations flexible.
- Current blurb, manuscript, outline, or chapter synopsis. If manuscript is unfinished, use positioning language instead of plot-specific claims.
- Available proof: reviews, awards, bestseller status, read-through data, series count, newsletter/freebie ecosystem, or editorial praise.
- Design destination: Canva, professional designer brief, direct image prompts, or copy-only package.

Ask only for missing information that materially changes the A+ structure. Otherwise make reasonable assumptions and label them.

## Workflow

1. **Determine readiness**
   - Cover finalized + manuscript unfinished: produce visual direction, module map, and placeholder copy.
   - Cover finalized + manuscript/blurb finished: produce final copy and designer-ready specs.
   - No final cover: produce strategy, comp trends, and flexible visual territories.

2. **Check market fit**
   - For current genre trends or comp-page references, browse Amazon and author/store pages when the user wants trend checking, comps, or "what is working now."
   - Focus on pattern recognition: module order, visual density, color mood, product/series emphasis, quote usage, comparison charts, trope/setting presentation, and mobile readability.
   - If browsing is unavailable, state that the trend read is based on genre heuristics and prior examples.

3. **Choose the A+ selling job**
   Pick one primary job and one secondary job:
   - Series credibility/read-through
   - Single-book premise and stakes
   - Character or relationship promise
   - Setting/world atmosphere
   - Trope/package clarity
   - Social proof and authority
   - Format/box-set value

4. **Build the module map**
   Use 4-7 modules for standard book A+ unless the user requests a larger package. Recommend fewer, stronger modules for mobile-heavy shoppers.

5. **Write copy**
   Use short, high-contrast copy that can survive on image modules. Avoid long paragraphs, tiny text, and claims that need substantiation.
   - Headlines: 3-8 words.
   - Image text blocks: usually 12-35 words.
   - Designer notes: state mood, background, crop, text placement, contrast, and which cover elements to echo.

6. **Prepare deliverables**
   Include a clear production package:
   - Strategy summary
   - Module-by-module layout
   - Final or draft copy
   - Visual direction and image prompts
   - Designer/Canva notes
   - Mobile readability warnings
   - Missing assets or decisions

## Module Defaults

For thrillers, suspense, mystery, and crime:
- Top series banner with book covers when the series has 3+ books.
- Cinematic title/stakes banner.
- Short premise or protagonist-pressure panel.
- Atmosphere/setting danger panel.
- Series/read-through or product mockup panel.
- Optional comparison chart for multiple books or series entry points.

For romance:
- Trope/promise banner.
- Couple or emotional tension panel.
- Setting/community panel if relevant.
- Quote-style longing/conflict panel.
- Series or linked-world panel.
- Heat/clean/faith tone should be implied carefully and genre-appropriately.

For cozy mystery:
- Series/world banner.
- Sleuth + community setup.
- Puzzle/stakes panel without becoming too dark.
- Recurring cast/pet/shop/theme panel if applicable.
- Series order or comparison chart.

For nonfiction:
- Authority/problem-solution banner.
- Outcome framework.
- What readers will learn.
- Credibility/proof.
- Format or workbook/bonus value.

Read [module-patterns.md](references/module-patterns.md) when the task needs more detailed module menus or genre-specific variations.

```markdown
**Blurb/Copy Notes**

```

When producing final designer specs, include dimensions only if the user or platform requires specific templates; otherwise keep the direction module-based so the designer can adapt to Amazon's current A+ interface.
