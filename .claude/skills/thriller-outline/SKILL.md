# Thriller Outline Template

Guide the user through an interactive, step-by-step process to develop a thriller concept into a fully detailed chapter outline. This workflow is specifically designed for thrillers and their subgenres. Every step pauses for user input before proceeding.

## Core Principles

- **One step at a time**: Complete each step fully, present the output, and WAIT for user feedback.
- **Market-informed**: Use K-Lytics data if uploaded. For suspense thrillers, ALWAYS read `references/reader-radar-suspense-thriller.md` first — it contains current demand snapshots, element rankings, comp title analysis, cast archetypes, market gaps, and risk assessments specific to the suspense thriller lane. Fall back to `references/thriller-market-data.md` for general thriller subgenres not covered by Reader Radar.
- **Information-control first**: Thrillers are about WHAT THE READER KNOWS AND WHEN THEY KNOW IT. The outline must track information release with precision.
- **Twist-proof**: Every twist must be reverse-engineerable — the reader should be able to go back and see the seeds. The outline maps twist logic explicitly.
- **Tension never flatlines**: Every chapter must either escalate tension, provide a brief relief valve (that sets up the next escalation), or deliver a revelation. No neutral chapters.
- **Fresh and original**: All names, places, and titles must avoid overused AI patterns.

## Names & Places Avoidance Rules

ALL generated content must avoid AI patterns. See `references/workflow-steps.md` for the full avoidance list. No "Sarah/Jack" generic thriller names. No "Blackwood Estates." No "The [Dark Adjective] [Noun]" titles.

## Number & Time Diversity (Anti-7 Bias)

Actively avoid 7 in all content — addresses, case numbers, years, ages, timestamps, countdown timers. Distribute across all digits.

## Reader Radar Integration (Suspense Thriller)

When the user selects **suspense thriller** (or any subgenre with strong suspense elements), read `references/reader-radar-suspense-thriller.md` and apply throughout the outline:

- **Story Idea Generation (Step 3)**: Prioritize the HIGH-demand elements from the Element Ranking — relentless pacing, a devastating secret/lie, personal stakes, and a late-act narrative-reframing twist. Flag any cooling trends the user's concept touches (global conspiracy, forensic procedural without personal stakes, tech-heavy exposition). Surface market gaps as differentiation opportunities.
- **Protagonist Design (Step 5)**: Default to the "Ordinary Person Who Discovers the Lie" archetype — the Reader Radar data confirms this outperforms trained-operative setups. If the user wants a different protagonist type, that's fine, but ensure personal stakes remain the driver.
- **Antagonist Design (Step 6)**: Default to the "Hidden Threat with a Personal Connection" — someone the protagonist trusted or someone connected to a person they trusted. Stranger-danger is less effective in this lane.
- **Supporting Cast (Step 9)**: Use the Reader Radar cast archetypes as a starting framework — The Person Who Knows Too Much, The Unexpected Ally, The Loved One at Risk.
- **Settings (Step 10)**: Favor contained/claustrophobic settings (MEDIUM demand, LOW supply = opportunity). Small-town or suburban pressure cooker is the proven model.
- **Chapter Outline (Step 11)**: Enforce the #1 ranked element — relentless forward-momentum pacing with chapter-end hooks. Short chapters (8-12 pages). Every chapter ends on a question, revelation, or threat escalation. Flag any pacing valley risk in the second act.
- **Series Strategy**: If the user is planning a series, recommend the series-connected-standalone model (FAST rising trend) — each book works alone but rewards loyal readers.
