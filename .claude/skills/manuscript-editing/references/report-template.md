# Editorial Report Template

Use this structure when generating the final editorial report (.docx) after all selected editing passes are complete.

## Document Formatting

- Title page: "[Manuscript Title] — Editorial Report"
- Subtitle: "Professional Fiction Edit with Track Changes"
- Author name and date
- Note which passes were run (e.g., "Passes completed: Line Edit, Proofread, Continuity Audit")
- Use heading levels for sections
- Before/after examples should be in a two-column format or clearly labeled blocks
- Use tables for quantitative data (counts, frequencies)

---

## Report Structure

### 1. Title Page

```
[MANUSCRIPT TITLE]
Editorial Report — 3-Pass Professional Edit

Author: [Author Name]
Editor: Manuscript Editor Pro
Date: [Date]
Original Word Count: [N]
Final Word Count: [N]
Genre: [Genre]
```

### 2. Executive Summary

Write 2-3 paragraphs covering:

**Paragraph 1 — Overall Assessment:**
What's the manuscript's strongest quality? What genre expectations does it meet well? What is the overall reading experience?

**Paragraph 2 — Key Issues Found:**
What were the most significant problems addressed across the three passes? Frame these constructively — not "your dialogue was terrible" but "dialogue distinctiveness was the area with the most room for growth, particularly in Chapters 5-9 where secondary characters shared similar speech patterns."

**Paragraph 3 — Trajectory:**
How much did the manuscript improve across the three editing passes? What's the overall arc from input to output? Is it ready for publication, or are there remaining recommendations?

### 3. Developmental Assessment Summary (if run)

**Note:** The developmental assessment is advisory only. No automatic prose changes were made. All suggestions are for the author to evaluate and implement at their discretion.

**Issues Identified:**
List all developmental issues found, organized by severity (Critical / Important / Minor), with chapter references and brief descriptions.

**Suggested Approaches:**
For the top 5 most significant issues, summarize the suggested approach provided in the full Developmental Assessment document.

**Full Assessment:** Refer to the separate "[Title] - Developmental Assessment.docx" document for complete details.

### 5. Continuity Audit Report

**Character Bible:**
Include the full character reference table — every named character with physical description, role, relationships, and first appearance. Authors can use this as a series bible for future books.

**Relationship Tracker:**
How and when characters meet, relationship progression across the manuscript. Note any relationship jumps that were smoothed during editing.

**Location & Geography Reference:**
All named locations with descriptions and spatial relationships. Include a simplified "map" of key locations if the story is set in one primary location (town, building, campus, etc.).

**Timeline Reconstruction:**
Day-by-day (or scene-by-scene) timeline of the manuscript's events.

**World Rules Summary (if applicable):**
All established rules, systems, and their sources within the text.

**Contradictions Found and Resolved:**

| Issue | Location | Before | After | Category |
|-------|----------|--------|-------|----------|
| [Description] | Ch. X, Ch. Y | [Original] | [Resolution] | Character / Relationship / Geography / Timeline / World Rule / Object |

The continuity tables are among the most valuable parts of this report — authors can use them as a reference bible for future books in a series, saving hours of re-reading their own work.

### 7. Per-Chapter Coverage Table (REQUIRED)

**This table is MANDATORY for every editorial report.** It holds the editing accountable by showing exactly how many issues were found in each chapter across each pass that was run. If chapters show zero issues, the author can see whether the editor was thorough or skimmed.

```
| Chapter | Line Edit Issues | Proofread Issues | Continuity Issues |
|---------|-----------------|-----------------|-------------------|
| Ch. 1   | 8               | 4               | 2                 |
| Ch. 2   | 12              | 6               | 1                 |
| Ch. 3   | 6               | 3               | 0                 |
| ...     | ...             | ...             | ...               |
| TOTAL   | N               | N               | N                 |
```

Only include columns for passes that were run. If only proofreading was selected, the table has Chapter + Proofread Issues columns.

### 8. Recommendations for the Author

This section is the editor's letter to the author. Write it in second person ("you"), warmly and constructively. Cover:

**Patterns to Watch:**
- 2-3 recurring issues the author should be aware of in future writing
- Frame as craft development, not criticism ("You tend to reach for hedging language in emotional scenes — a sign that you're not yet trusting yourself to commit to the moment. As you grow in confidence, you'll find those 'seemed to' constructions disappearing naturally.")

**Strengths to Lean Into:**
- 2-3 specific things the author does exceptionally well
- Be specific — not "good dialogue" but "your use of silence in dialogue is remarkable — the unspoken tension in the confrontation scene (Ch. 8) was the strongest moment in the manuscript"

**Genre Observations:**
- How well does the manuscript fit current market expectations for its genre?
- Any suggestions for strengthening genre appeal?

**Next Steps:**
- Is the manuscript ready for publication?
- If not, what additional work is recommended?
- Suggest beta readers, sensitivity readers, or other review if appropriate

### 9. How to Review Your Edited Manuscript

Include clear instructions for the author:

1. **Open the EDITED.docx file** in Microsoft Word, Google Docs, or LibreOffice Writer
2. **Turn on Track Changes view** (in Word: Review tab → All Markup; in Google Docs: View → Show suggested edits)
3. **Review each change** — you'll see deletions (usually in red strikethrough) and insertions (usually in colored underline)
4. **Accept or reject each change** individually:
   - In Word: Right-click a change → Accept/Reject, or use the Review tab arrows
   - In Google Docs: Click the checkmark (accept) or X (reject) on each suggestion
5. **Read the comments** — some changes include editor comments explaining the reasoning
6. **Save the final version** after reviewing all changes

**You are in complete control.** Every change is a suggestion. Accept what improves your book, reject what doesn't fit your vision.
